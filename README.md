# CashTrack v12

## תיקון מעבר חודש

בפתיחת חודש חדש, יתרת הפתיחה מחושבת כעת כך:

`יתרת הפתיחה בחודש החדש = היתרה הצפויה בסוף החודש הקודם`

החישוב כולל:

- יתרת העו״ש של החודש הקודם
- נטו משוער של המשכורת
- הכנסה נוספת נטו
- חיוב אשראי כולל
- הוראות קבע, תשלומים קבועים והוצאות קבועות

היתרה מסומנת כמשוערת עד שהמשתמש שומר יתרה מעודכנת לפי חשבון הבנק. לאחר שמירה ידנית היא לא תוחלף שוב אוטומטית.

בנוסף:

- תקציב חודשי שהוגדל, למשל ל־2,500 ₪, ממשיך לחודש הבא.
- סכומים חודשיים כמו אשראי, הכנסה נוספת וניצול התקציב מתאפסים.
- הוראות קבע, הגדרות שכר והיסטוריית חודשים נשמרות.
- האפליקציה מזהה מעבר חודש גם אם נשארה פתוחה בלילה.
- מספר הימים בחודש נקבע אוטומטית, כולל פברואר ושנה מעוברת.
